## نظرة عامة

يهدف هذا المشروع إلى توفير واجهة سهلة للطلاب للاستعلام عن المواد الدراسية المقررة عليهم وتفاصيل الامتحانات (اللجنة، القاعة، التاريخ، الفترة)، مع إمكانية تصدير النتيجة كصورة. كما يقدم لوحة تحليلات (Dashboard) تعرض إحصائيات شاملة مثل عدد الطلاب، عدد المقررات، توزيع المساقات، وأكثر من رسم بياني تفاعلي باستخدام مكتبة Plotly.

يتم توليد ملفات البيانات (JSON) بشكل شبه آلي من ملفات نصية (txt) تحتوي على بيانات الطلاب والمواد، مما يسهل تحديث النظام في بداية كل فصل دراسي.

---

## الميزات الرئيسية

- **البحث المتقدم**: بالاسم أو الرقم الأكاديمي، مع دعم التشكيل والبحث الجزئي.
- **نتائج مفصلة**: تعرض قائمة المواد مع رابط لمجلد المادة على Google Drive (إن وُجد)، وتفاصيل الامتحان.
- **تصدير الصورة**: تحويل بطاقة المواد إلى صورة PNG وحفظها على الجهاز.
- **عداد الزوار**: تسجيل عدد مرات البحث الفريدة عبر `counter.php`.
- **سجل البحث محلياً**: حفظ آخر 10 عمليات بحث في `localStorage` مع إمكانية المسح.
- **لوحة تحليلات (Dashboard)**:
  - بطاقات إحصائية: إجمالي الطلاب، عدد المقررات، متوسط المقررات لكل طالب، المقرر الأعلى تسجيلاً.
  - رسوم بيانية: المقررات الأكثر تسجيلاً (Bar Chart)، توزيع المقررات (Pie Chart)، توزيع عدد المقررات لكل طالب (Histogram)، صندوق الإحصاء (Box Plot).
  - قائمة بأكثر 10 مقررات شعبية مع أشرطة تقدم تفاعلية.
- **جاهزية البيانات**: يتم إنشاء جميع ملفات JSON تلقائياً من النصوص المصدرية باستخدام سكريبتات Python.

## هيكل المشروع

```

│   └── jsonOutput.py         # تجميع ملفات اللجان وتحويلها إلى JSON

│   ├── rooms.json            # قائمة القاعات المستخدمة
│   └── students.json         # بيانات الطلاب (الرقم، الاسم، قائمة المقررات)
├── foldersData/              # المخرجات الوسيطة من foldersOutput.py (مجلدات حسب الأقسام)
│   ├── CS/
│   ├── IT/
│   ├── IS/
│   └── gen/
├── txtData/                  # الملفات النصية المصدرية (يتم تحريرها يدوياً)
│   ├── CS.txt                # طلاب قسم CS
│   ├── IT.txt                # طلاب قسم IT
│   ├── IS.txt                # طلاب قسم IS
│   ├── gen.txt               # طلاب عام (مواد مشتركة)
│   └── subjects.txt          # قائمة المواد مع أكوادها
├── index.php                 # واجهة البحث الرئيسية
├── search.php                # نقطة نهاية البحث (API)
├── counter.php               # عداد الزوار (زيادة وقراءة)
├── counter.txt               # ملف تخزين قيمة العداد
├── dashboard.html            # لوحة التحليلات (HTML/JS + Plotly)
├── dashboard-icon.png        # أيقونة زر Dashboard
├── favicon.ico               # أيقونة الموقع
├── students.json             # (نسخة احتياطية قديمة) يمكن تجاهلها
└── getInfo/                  # سكريبتات إضافية (غير مستخدمة في التشغيل الأساسي)
    ├── getStdJson.py
    ├── getStd.py
    └── subjects.py
```

## كيفية الاستخدام

### 1. البحث عن الطالب

- اكتب الاسم (بأي تشكيل) أو الرقم الأكاديمي في حقل البحث.
- ستظهر النتائج تلقائياً بعد توقف الكتابة (Debounce).
- انقر على أي نتيجة سابقة من **سجل البحث** لاستخدامها مجدداً.
- اضغط على زر **📸 تحميل صورة المواد** لتصدير بيانات الطالب كصورة PNG.

### 2. لوحة التحليلات

- انتقل إلى `dashboard.html` يدوياً أو اضغط على زر **Dashboard** الموجود أعلى يسار صفحة البحث.
- تعرض اللوحة إحصائيات آنية مستمدة من ملف `data/students.json` و `data/courses.json`.
- يمكن تحديث اللوحة يدوياً بزر التحديث ↻ أعلى الصفحة.

### 3. عداد الزوار

- يسجل العداد كل عملية بحث ناجحة (أول مرة فقط لكل جلسة بحث).
- يتم تحديث العداد بشكل دوري كل 5 ثوانٍ.

## تحديث البيانات

عند بداية فصل دراسي جديد (تغيير في المواد، الطلاب، أو جداول الامتحانات):

1. **تعديل الملفات النصية المصدرية** في مجلد `txtData` وفق نفس التنسيق:  
   - كل مادة تبدأ بسطر مثل: `(CS438) المقرر برمجة تطبيقات المحمول`  
   - أسفلها قائمة الطلاب المسجلين، كل طالب في سطر منفصل بصيغة `الاسم\tالرقم` أو `الاسم الرقم`.

2. **تعديل بيانات الامتحانات والقاعات** داخل `foldersOutput.py` (القواميس `raw_exam_schedule` و `raw_room_schedule`).

3. **إعادة تشغيل سكريبتات Python** بنفس الترتيب:
   ```bash
   python codePyData/foldersOutput.py
   python codePyData/jsonOutput.py
   ```

4. **تحديث ملفات JSON** – سيتم استبدال الملفات الموجودة في مجلد `data` تلقائياً.

5. **حذف سجل البحث المحلي** (اختياري) – من واجهة المستخدم بزر "مسح السجل" أو يدوياً من `localStorage`.

## التقنيات المستخدمة

- **PHP**: نقاط نهاية خفيفة (البحث والعداد).
- **JavaScript (ES6+)**: المنطق الأمامي بالكامل، تضمين السجل المحلي، التصدير كصورة باستخدام `html2canvas`.
- **Python**: معالجة البيانات النصية وتحويلها إلى JSON (باستخدام `os` و `json` و `re`).
- **Plotly.js**: إنشاء الرسوم البيانية التفاعلية في لوحة التحليلات.
- **Google Fonts**: خطوط Sora, DM Sans, DM Mono, Cairo.
- **CSS**: تصميم عصري بخلفيات داكنة، تأثيرات زجاجية، وأيقونات.
